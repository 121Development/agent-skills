# Make a README

A step-by-step guide for writing your first README or refreshing a stale one.

## Before You Start

1. Know your audience (see `references/art-of-readme.md`)
2. Pick your project type (see main skill)
3. Have the project running locally — you can't document what you haven't tested

## Step 1: The Hook

Write one sentence that says what it does and why someone should care.

Bad: "A CLI tool"
Good: "Scaffolds Node projects with linting, testing, and CI pre-configured"

## Step 2: The Quick Start

Show the fastest path from "I found this" to "it works."

```bash
npm install -g my-tool
my-tool init my-project
cd my-project
npm start
```

## Step 3: The Example

Show a realistic use case, not a toy one. Include expected output.

## Step 4: The Details

Add sections as needed: configuration, API, troubleshooting, contributing.

## Step 5: The Review

Read it as if you've never seen the project. What's missing? What's confusing?
