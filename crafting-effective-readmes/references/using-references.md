# Using Reference Materials

This skill includes several layers of reference. Use them appropriately:

## Quick Reference (loaded with skill)

- **`references/section-checklist.md`** - When you need to know "what sections should this README have?"
- **`references/style-guide.md`** - When you need to fix prose or avoid common mistakes

## Templates (loaded on demand)

- **`templates/oss.md`** - Starting point for open source projects
- **`templates/personal.md`** - Starting point for personal projects
- **`templates/internal.md`** - Starting point for internal/team projects
- **`templates/xdg-config.md`** - Starting point for config/dotfiles

Load the template that matches the project type, then adapt.

## Deep Reference (for learning)

- **`references/art-of-readme.md`** - Philosophy and principles
- **`references/make-a-readme.md`** - Step-by-step tutorial
- **`references/standard-readme-spec.md`** - Formal specification

These are for understanding *why*, not for copying into a project README.

## When to use what

| Situation | Reach for |
|-----------|-----------|
| Need a starting structure | Template |
| Not sure what sections to include | Section checklist |
| Writing sounds bad | Style guide |
| Want to understand README philosophy | Deep references |
| Full example needed | `references/standard-readme-example-*.md` |
