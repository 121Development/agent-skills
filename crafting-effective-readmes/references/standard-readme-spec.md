# Standard README Specification

A formal specification for README structure, based on common conventions across open source projects.

## Required Elements

1. **Title** - The project name
2. **Description** - What it does
3. **Installation** - How to get it
4. **Usage** - How to use it
5. **License** - Legal terms

## Optional Elements

- Badges (build status, version, etc.)
- Table of contents (for long READMEs)
- API documentation
- Contributing guidelines
- Changelog
- Acknowledgements

## Ordering

Follow the inverted pyramid: most important first. A reader should get value from the first three sections even if they stop there.
